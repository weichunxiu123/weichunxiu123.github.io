Fume安装及测试

一、Flume安装

1、Flume安装

（1）上传flume的压缩包到/export/software下。

（2）解压到/export/server目录下。

tar -zxvf apache-flume-1.9.0-bin.tar.gz -C /export/server/

同时设置软链接：

ln -s /export/server/apache-flume-1.9.0-bin/ flume

如下图所示：

（3）编辑/etc/profile，配置FLUME_HOME指向正确的路径

#FLUME_HOME
export FLUME_HOME=/export/server/flume
export PATH=$PATH:$FLUME_HOME/bin
source /etc/profile

（4）通过scp命令发送给其余机器，并设置软链接。

scp -r /export/server/apache-flume-1.9.0-bin root@node1:/export/server/
scp -r /export/server/apache-flume-1.9.0-bin root@node3:/export/server/
ln -s /export/server/apache-flume-1.9.0-bin /export/server/flume

（5）添加配置文件

在flume/myconf目录下添加配置文件netcat-logger.conf，添加如下内容：

# example1-netcat-logger.conf: 单节点Flume配置
\# 定义agent名称为a1
\# 设置3个组件的名称
a1.sources = r1
a1.sinks = k1
a1.channels = c1
\# 配置source类型为NetCat,监听地址为本机，端口为44444
a1.sources.r1.type = netcat
a1.sources.r1.bind = 0.0.0.0
a1.sources.r1.port = 44444
\#source和channel关联
a1.sources.r1.channels = c1
\# 配置channel类型为内存，内存队列最大容量为1000，一个事务中从source接收的Events数量或者发送给sink的Events数量最大为100
a1.channels.c1.type = memory
a1.channels.c1.capacity = 1000
a1.channels.c1.transactionCapacity = 100
\# 配置sink类型为Logger
a1.sinks.k1.type = logger
\# 将sink绑定到channel上
a1.sinks.k1.channel = c1

（6）启动flume

在/export/server/flume目录下：

bin/flume-ng agent -n a1 -c conf -f myconf/example1-netcat-logger.conf -Dflume.root.logger=INFO,console

使用Netcat测试，复制node1标签，启动netcat连接到44444端口

可以看到agent控制台接收到信息。

二、Flume测试

（1）exec_source测试

使用Flume从日志文件中将日志收集到日志中间，以便于查找和分析。启动测试流程为：

1）准备日志文件 

mkdir -p /export/data/flume-example-data

2）写一个脚本模拟向日志文件中持续写入数据，

mkdir -p /export/data/flume-example-data/shell；vim access_data.sh
for i in {1..10000}；
do echo ${i} “bigdata log 5/11” >> access.log;
sleep 0.5;
done

3）创建flume自定义配置文件，放到/export/server/flume/myconf目录下。

# example2-exec-source-logger.conf
a1.sources = r1
a1.channels = c1
a1.sinks = k1
a1.sources.r1.type = exec
a1.sources.r1.channels = c1
a1.sources.r1.command = tail -F /export/data/flume-example-data/shell
/access.log 
a1.sources.r1.batchSize = 100
a1.channels.c1.type = memory
a1.channels.c1.capacity = 1000
a1.channels.c1.transactionCapacity = 100
a1.sinks.k1.type = logger
a1.sinks.k1.channel = c1

4）启动flume

bin/flume-ng agent -n a1 -c conf/ -f myconf/example2-exec-source-logger.conf -Dflume.root.logger=INFO,console

（2）spooldir_source测试

监听一个指定的文件夹，如果文件夹下有没有采集过的新文件，则会采集新文件中的数据。监听测试流程为：

1）创建spooldir监听目录。

mkdir -p /export/data/flume-example-data/spooldir

2）在spooldir下新建文件。

mkdir flume511.txt

3）创建flume自定义配置文件，放到/export/server/flume/myconf目录下。

#example3-spooldir-source.conf
a1.sources = r1
a1.channels = c1
a1.sinks = k1
a1.sources.r1.channels = c1
a1.sources.r1.type = spooldir
a1.sources.r1.spoolDir = /export/data/flume-example-data/spooldir
a1.channels.c1.type = memory
a1.channels.c1.capacity = 1000
a1.channels.c1.transactionCapacity = 100
a1.sinks.k1.type = logger
a1.sinks.k1.channel = c1

4）启动flume

bin/flume-ng agent -n a1 -c conf -f myconf/example3-spooldir-source.conf -Dflume.root.logger=INFO,console

（3）taildir_source测试

监听指定目录下的文件，只要文件中有新写入的行，就会被tail到，它会记录每一个文件所tail到的位置，记录到一个指定的positionfile保存目录中，格式为json。保证数据不会漏采（丢失）。 测试流程如下：

1）新建监听目录 

mkdir -p /export/data/flume-example-data/weblog/web-access.log
mkdir -p /export/data/flume-example-data/weblog/web-access.log.1
mkdir -p /export/data/flume-example-data/wxlog/wx-access.log

2）生成测试数据

for i in {1..1000}
do
echo "webweb111 $RANDOM" >> /export/data/flume-example-data/
/web-access.log
echo "webweb222 $RANDOM" >> /export/data/flume-example-data/
log/web-access.log.1
echo "wxwxwx333 $RANDOM" >> /export/data/flume-example-data
/wxlog/wx-access.log
sleep 0.1
done

3）创建flume自定义配置文件，放到/export/server/flume/myconf目录下。

#example4-taildir-source.conf
a1.sources = r1
a1.sinks = k1
a1.channels = c1
a1.sources.r1.type = TAILDIR
a1.sources.r1.channels = c1
a1.sources.r1.positionFile = /export/data/flume-example-data/flumedata/taildir_position.json
a1.sources.r1.filegroups = g1 g2
a1.sources.r1.filegroups.g1 = /export/data/flume-example-data/weblog/web.*
a1.sources.r1.filegroups.g2 = /export/data/flume-example-data/wxlog/wx.*
a1.sources.r1.fileHeader = true
\#动态的header-keys eg：filepath=/../../../
a1.sources.r1.fileHeaderKey = filepath
\#写死的header-keys（静态的） eg:a1 = aa1
a1.sources.r1.headers.g1.a1 = aa1
a1.sources.r1.headers.g1.b1 = bb1
a1.sources.r1.headers.g2.a2 = aa2
a1.sources.r1.headers.g2.b2 = bb2
a1.sources.r1.maxBatchCount = 1000
a1.channels.c1.type = memory
a1.channels.c1.capacity = 10000
a1.channels.c1.transactionCapacity = 1000
a1.sinks.k1.type = logger
a1.sinks.k1.channel = c1

4）启动flume

bin/flume-ng agent -n a1 -c conf/ -f  myconf/example4-taildir-source.conf -Dflume.root.logger=INFO,console

（4）avro source测试

avro source 是通过监听一个网络端口来接收数据，被接收数据必须是使用avro序列化框架序列化后的数据。

工作机制：启动一个网络服务，监听一个端口，收集端口上收到的avro序列化数据流。

测试流程如下：

1）自定义创建flume自定义配置文件，放到/export/server/flume/myconf目录下。

#example5-avro-source.conf
a1.sources = r1
a1.sources.r1.type = avro
a1.sources.r1.channels = c1
a1.sources.r1.bind = 0.0.0.0
a1.sources.r1.port = 4141
a1.channels = c1
a1.channels.c1.type = memory
a1.channels.c1.capacity = 200
a1.channels.c1.transactionCapacity = 100
a1.sinks = k1
a1.sinks.k1.type = logger
a1.sinks.k1.channel = c1

2）启动agent

bin/flume-ng agent -c conf -f  myconf/example5-avro-source.conf -n a1 -Dflume.root.logger=INFO,console

注意：在node1的/export/server/flume目录下执行。

3）新建avro-log.txt，用一个客户端去给启动好的source发送avro序列化数据

mkdir -p /export/data/flume-example-data/avro-log.txt  
bin/flume-ng avro-client --host node1  --port 4141  -f /export/data/flume-example-data/avro-log.txt

注意：复制标签在新窗口执行！！！

最后在agent控制台可以看到avro-log.txt内的内容。

（5）使用File Channel实现数据持久化

测试流程如下：

1）自定义flume配置文件，并放到/export/server/flume/myconf中

#example6-file-channel.conf
\# 定义agent名称为a1
\# 设置3个组件的名称
a1.sources = r1
a1.sinks = k1
\# 多个channel使用空格分隔
a1.channels = c1 c2
\# 配置source类型为NetCat,监听地址为本机，端口为44444
a1.sources.r1.type = netcat
a1.sources.r1.bind = 0.0.0.0
a1.sources.r1.port = 44444
\# 配置sink类型为Logger
a1.sinks.k1.type = logger
\# 配置channel类型为内存，内存队列最大容量为1000，一个事务中从source接收的Events数量或者发送给sink的Events数量最大为100
a1.channels.c1.type = memory
a1.channels.c1.capacity = 1000
a1.channels.c1.transactionCapacity = 100
\# 配置FileChannel,checkpointDir为检查点文件存储目录，dataDirs为日志数据存储目录，
a1.channels.c2.type = file
a1.channels.c2.checkpointDir = /export/data/flume-example-data/flumedata/checkpoint_filechannel
a1.channels.c2.dataDirs = /export/data/flume-example-data/flumedata/data_filechannel
\# 将source和sink绑定到channel上
\# source同时绑定到c1和c2上
a1.sources.r1.channels = c1 c2
a1.sinks.k1.channel = c1 | c2

2）为了方便日志打印，可以将-Dflume.root.logger=INFO,console添加在conf的环境配置中。

cp flume-env.sh.template flume-env.sh
vi flume-env.sh
# 添加JAVA_OPTS
export JAVA_OPTS="-Dflume.root.logger=INFO,console"

2）启动flume

bin/flume-ng agent -n a1 -c conf -f myconf/example6-file-channel.conf -Dflume.root.logger=INFO,console

（6）利用avro source和avro sink实现agent级联

（1）基本介绍：可以将多个Flume agent 程序连接在一起，其中一个agent的sink将数据发送到另一个agent的source。从多个Web服务器收集日志，发送到一个或多个集中处理的agent，之后再发往日志存储中心。

（2）测试流程如下：

\1) 启动hdfs，并检查工作状态。node1上有DataNode,NodeManager,Namenode,ResourceManager。node2上有NodeManager,SecondaryNameNode,DataNode。node3上有NodeManager,DataNode。

\2) 配置上游配置文件，保存到node1,node2上。

\3) 配置下游配置文件，保存到node3上。

\4) 启动下游node3上的flume agent。

bin/flume-ng agent -n a1 -c conf/ -f myconf/example7-2-avro-f-hdfs.conf -Dflume.root.logger=DEBUG，console

\5) 在node1和node2上准备两个日志目录来生成模拟日志数据。

mkdir /export/data/flume-example-data/weblog/
mkdir /export/data/flume-example-data/wxlog/

\6) 在node1和node2上利用shell脚本生成日志数据。

vim avro-hdfs.sh
while true
do
echo webwebwebwebweb >> /export/data/flume-example-data/weblog
/web-access.log
echo wxwxwxwxwxwxwx >> /export/data/flume-example-data/wxlog/
wx-access.log
sleep 0.01
Done

7）启动node1和node2上的flume agent

nohup bin/flume-ng agent -n a1 -c conf/ -f myconf/example7-1-taildir-f-avro.conf -Dflume.root.logger=INFO,console 1>/dev/null 2>&1 &

8）上游配置文件：

#上游服务器配置 example7-1-taildir-f-avro.conf

a1.sources = r1
a1.channels = c1
a1.sinks = k1
a1.sources.r1.type = TAILDIR
a1.sources.r1.channels = c1
a1.sources.r1.positionFile = /export/data/flume-example-data/flumedata/taildir_position.json
a1.sources.r1.filegroups = g1 g2
a1.sources.r1.filegroups.g1 = /export/data/flume-example-data/weblog/web.*
a1.sources.r1.filegroups.g2 = /export/data/flume-example-data/wxlog/wx.*
\#提高吞吐量
a1.sources.r1.batchSize = 1000
\#动态的header-keys eg：filepath=/../../../
a1.sources.r1.fileHeaderKey = filepath
\#拦截器配置，添加header=timestamp
a1.sources.r1.interceptors = i1
a1.sources.r1.interceptors.i1.type = timestamp
a1.sources.r1.interceptors.i1.headerName = timestamp
a1.channels.c1.type = file
\#本机数据汇集检查点、event存储目录
a1.channels.c1.checkpointDir = /export/data/flume-example-data/flumedata/checkpoint
a1.channels.c1.dataDirs = /export/data/flume-example-data/flumedata/data
a1.channels.c1.transactionCapacity = 2000
a1.sinks.k1.channel = c1
a1.sinks.k1.type = avro
a1.sinks.k1.batch-size = 1000
\#下游目标主机、端口
a1.sinks.k1.hostname = node3
a1.sinks.k1.port = 44444

9）下游配置文件：

#下游服务器配置 example7-2-avro-f-hdfs.conf

a1.sources = r1
a1.channels = c1
a1.sinks = k1
#下游数据汇集avro source
a1.sources.r1.type = avro
a1.sources.r1.channels = c1
a1.sources.r1.bind = 0.0.0.0
a1.sources.r1.port = 44444
a1.sources.r1.threads = 10
a1.sources.r1.batchSize = 1000 
a1.channels.c1.type = file
a1.channels.c1.checkpointDir = /export/data/flume-example-data/flumedata/checkpoint
a1.channels.c1.dataDirs = /export/data/flume-example-data/flumedata/data
a1.channels.c1.transactionCapacity = 2000 
#hdfs sink
a1.sinks.k1.channel = c1
a1.sinks.k1.type = hdfs
a1.sinks.k1.hdfs.path = hdfs://node1:8020/logdata/%Y-%m-%d/%H/
#eg：文件名 logdata_34438hxfd.log，在滚动时，logdata_34438hxfd.log.tmp
a1.sinks.k1.hdfs.filePrefix = logdata_
a1.sinks.k1.hdfs.fileSuffix = .log
#三个条件没有优先级，谁先达到就进行滚动
#按时间间隔滚动
a1.sinks.k1.hdfs.rollInterval = 0
#按文件大小滚动 256MB
a1.sinks.k1.hdfs.rollSize = 268435456
#按event条数滚动
a1.sinks.k1.hdfs.rollCount = 100000
a1.sinks.k1.hdfs.batchSize = 1000
a1.sinks.k1.hdfs.codeC = gzip
a1.sinks.k1.hdfs.fileType = CompressedStream

先在下游启动flume agent 。下游的agent的sink将数据发送到上游agent的source。
