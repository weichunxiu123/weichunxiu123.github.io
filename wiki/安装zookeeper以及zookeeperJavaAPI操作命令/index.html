zookeeper安装以及zookeeperJavaAPI操作命令

zookeeper安装

安装前需要安装好jdk，检测集群时间是否同步，检测防火墙是否关闭，检测主机 ip映射有没有配置

（1）在node1上切换到 /export/server 目录下，上传zookeeper压缩包并解压，设置一个软连接。

切换到server目录下：

cd /export/server

解压zookeeper压缩包：

tar -zxvf zookeeper.tar.gz -C /export/server/

设置软连接：

ln -s  /export/server/zookeeper-3.4.6/ zookeeper

\1. 修改环境变量（三台都修改）

vi /etc/profile
export ZOOKEEPER_HOME=/export/server/zookeeper
export PATH=$PATH:$ZOOKEEPER_HOME/bin
source /etc/profile

\2. 修改zookeeper配置文件

切换到conf/目录下

cd /export/server/zookeeper/conf/

复制zoo_sample.cfg文件，文件名为zoo.cfg

cp zoo_sample.cfg zoo.cfg

创建文件：

mkdir -p /export/data/zookeeper/zkdatas/   
vim zoo.cfg

填充以下内容：

#Zookeeper的数据存放目录
dataDir = /export/data/zookeeper/zkdatas/
# 保留多少个快照
autopurge.snapRetainCount = 3
# 日志多少小时清理一次
autopurge.purgeInterval = 1
# 集群中服务器地址
server.1 = node1:2888:3888
server.2 = node2:2888:3888
server.3 = node3:2888:3888

\3. 添加myid配置

在node1主机的/export/server/zookeeper/zkdatas/这个路径下创建一个文件，文件名为myid ,文件内容为1

echo 1 > /export/data/zookeeper/zkdatas/myid

\4. 安装包分发并修改myid的值

在node1主机上，将安装包分发到其他机器

第一台机器上面执行以下两个命令

cd /export/server/
scp -r /export/server/zookeeper-3.4.6/ root@node2:/export/server/

分发node2

scp -r /export/server/zookeeper-3.4.6/ root@node3:/export/server/

分发node3

ln -s zookeeper-3.4.6/ zookeeper  建立软连接（node2 node3）
echo 2 > /export/data/zookeeper/zkdatas/myid  （node2上执行）
echo 3 > /export/data/zookeeper/zkdatas/myid  （node3上执行）

\5. 三台机器启动zookeeper服务

/export/server/zookeeper/bin/zkServer.sh start

（2） 三台主机分别查看启动状态

/export/server/zookeeper/bin/zkServer.sh status

\6. 编写一个脚本批量启动node1，2，3的zookeeper

（1）创建shell目录

mkdir /export/shell

（2）再此目录下建立一个zkall.sh文件

vim zkall.sh

添加以下内容：

#!/bin/bash
case $1 in
"start"){
	for i in node1 node2 node3
do
		echo ---------- zookeeper $i 启动 ------------
		ssh $i "/export/server/zookeeper/bin/zkServer.sh start"
	done
};;

"stop"){
	for i in node1 node2 node3

	do

		echo ---------- zookeeper $i 停止 ------------ 

		ssh $i "/export/server/zookeeper/bin/zkServer.sh stop"

	done
};;

"status"){
	for i in node1 node2 node3

	do

		echo ---------- zookeeper $i 状态 ------------ 

		ssh $i "/export/server/zookeeper/bin/zkServer.sh status"

	done

};;
Esac

（3）配置zk脚本环境变量、zookeeper的环境变量

vi /etc/profile
#ZOOKEEPER_SHELL_HOME
export ZKS_HOME=/export/shell/
export PATH=$PATH:$ZKS_HOME
export ZK_HOME=/export/server/zookeeper
export PATH=${ZK_HOME}/bin:$PATH

让环境变量生效

source /etc/profile

（4） 设置环境路径

vim .bashrc

（5） 增加可执行权限

chmod +x zkall.sh

（6）启动测试

zkall.sh start

查看状态
zkall.sh status

（7）zookeeper服务器常用命令

启动 ZooKeeper 服务 ./zkServer.sh start
查看 ZooKeeper 服务状态 ./zkServer.sh status
停止 ZooKeeper 服务 ./zkServer.sh stop 
重启 ZooKeeper 服务 ./zkServer.sh restart

\7. Zookeerper命令操作

（1）连接ZooKeeper服务端

./zkCli.sh -server node1:2181

\8. ZooKeeper JavaAPI 操作

（1）建立连接

client = CuratorFrameworkFactory.builder()
        .connectString("192.168.88.161:2181")
        .sessionTimeoutMs(60 * 1000)
        .connectionTimeoutMs(15 * 1000
        .retryPolicy(retryPolicy)
        .namespace("bigdata")
        .build();
    client.start();

（2）创建节点（带有数据）

String path = client.create().forPath("/app2", "hehe".getBytes())
System.out.println(path);

（3）查询结点

byte[] data = client.getData().forPath("/app2")
System.out.println(new String(data));

（4）修改数据

client.setData().forPath("/app2", "bigdata".getBytes())

（5）删除节点

client.delete().forPath("/app1")

（6）Watch事件监听

TreeCache treeCache = new TreeCache(client,"/app2");
    treeCache.getListenable().addListener(new TreeCacheListener() {
      @Override
      public void childEvent(CuratorFramework client, TreeCacheEvent event) throws Exception {
        System.out.println("节点变化了");
        System.out.println(event);
     }
    });  
   treeCache.start();

（7）分布式锁实现

public static void main(String[] args) {
    Ticket12306_old ticket12306_old = new Ticket12306_old();
    Thread t1 = new Thread(ticket12306_old,"携程");
    Thread t2 = new Thread(ticket12306_old,"飞猪");
    t1.start();
    t2.start();
}
